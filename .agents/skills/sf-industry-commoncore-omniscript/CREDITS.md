## Key Concepts Credited

### OmniScript Architecture
The Type/SubType/Language triplet pattern, OmniProcess/OmniProcessElement data model, and PropertySetConfig JSON structure are documented in official Salesforce OmniStudio documentation.

### Element Design Patterns
Best practices for step design, conditional visibility, data prefill, and navigation patterns are derived from Salesforce OmniStudio documentation, Trailhead modules, and community implementation experience.

### Scoring Framework
The 120-point scoring system across 6 categories is original to this skill, inspired by the sf-flow scoring framework and adapted for OmniScript-specific concerns (data integration, element design, client-side security).

### Integration Patterns
OmniScript orchestration patterns with Integration Procedures, DataRaptors, and FlexCards are documented in Salesforce OmniStudio developer guides and refined through community implementation feedback.
