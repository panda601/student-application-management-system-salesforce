# Security Policy

We take the security of the Student Application Management System (SAMS) seriously. This file outlines our support policy and vulnerability reporting instructions.

## Supported Versions

Only the latest stable release of SAMS is actively supported for security updates:

| Version | Supported |
| :--- | :--- |
| v1.0.x | :white_check_mark: Yes |
| < v1.0.0 | :x: No |

## Reporting a Vulnerability

If you discover a security vulnerability in SAMS, please do **NOT** open a public issue on GitHub. Instead, report it privately to our security team.

### Submission Guidelines
*   Send a detailed email describing the vulnerability to: `security-sams@example.com`.
*   Include a Proof of Concept (PoC) or step-by-step reproduction guide.
*   We aim to acknowledge your report within 48 hours and provide a fix or mitigation plan within 14 business days.
*   Once a fix is released, we will credit you in the release notes unless you request to remain anonymous.
